# Web デザイン品質基準

## アンチテンプレートポリシー

いかにも汎用テンプレート風の UI を出さないでください。フロントエンドのアウトプットは、意図的で、主張があり、そのプロダクトに固有のものに見えるべきです。

### 禁止パターン

- 一様なスペーシングで階層のないデフォルトのカードグリッド
- センタリングされた見出し、グラデーションブロブ、ありきたりな CTA で構成されたストックのヒーローセクション
- ライブラリのデフォルトをそのまま完成デザインとして出す
- レイヤー、深み、モーションのないフラットなレイアウト
- すべてのコンポーネントで角丸、スペーシング、シャドウが均一
- グレー on ホワイトに装飾的なアクセントカラー 1 色だけの無難なスタイリング
- サイドバー + カード + チャートで視点のない「決まり切った」ダッシュボードレイアウト
- 意図なく使われたデフォルトのフォントスタック

### 必須の品質

意味のあるフロントエンドのサーフェスは、以下のうち少なくとも 4 つを満たすべきです：

1. スケールのコントラストによる明確な階層
2. スペーシングに意図的なリズムがあり、どこもかしこも同じパディングではない
3. オーバーラップ、シャドウ、サーフェス、モーションによる深みやレイヤー
4. 個性のあるタイポグラフィと本物のペアリング戦略
5. 装飾的にではなくセマンティックに使われる色
6. デザインされたと感じられる hover / focus / active 状態
7. 適切な場面でのグリッドを破る editorial / bento コンポジション
8. ビジュアルの方向性に合うテクスチャ、グレイン、アトモスフィア
9. 邪魔にならず流れを明確にするモーション
10. 後付けではなく、デザインシステムの一部として扱われるデータビジュアライゼーション

## フロントエンドコードを書く前に

1. 具体的なスタイルの方向性を選ぶ。「clean minimal」のような曖昧なデフォルトは避ける。
2. パレットを意図的に定義する。
3. タイポグラフィを意識的に選ぶ。
4. 少なくとも少数の実在するリファレンスを集める。
5. 関連する ECC のデザイン/フロントエンドスキルを利用する。

## 検討する価値のあるスタイルの方向性

- Editorial / magazine
- Neo-brutalism
- 実際の深みのある Glassmorphism
- コントラストを節度をもって効かせた Dark luxury / Light luxury
- Bento レイアウト
- Scrollytelling
- 3D の統合
- Swiss / International
- Retro-futurism

自動的にダークモードをデフォルトにしないでください。プロダクトが実際に求める視覚的方向性を選んでください。

- [ ] 意図のある hover / focus / active 状態を持っているか？
- [ ] 一様な強調ではなく階層を使っているか？
- [ ] 実際のプロダクトのスクリーンショットとして違和感なく見えるか？
- [ ] 両テーマをサポートするなら、ライトとダークの両方が意図的に感じられるか？
