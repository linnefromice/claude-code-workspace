# PR-Based Learning

マージ済みPRを分析し、`.claude/rules/` と `.claude/skills/` の更新提案を生成する。

## 使用方法

```bash
/pr-learning              # 過去7日間のマージ済みPRを分析
/pr-learning 14           # 過去14日間を分析
/pr-learning 30           # 過去30日間を分析
```

## 制約

- **読み取り専用**: ルール・スキルファイルの直接変更は行わない
- **提案ベース**: 分析結果は提案としてコンソールに出力する
- **最大PR数**: 20PRを上限とする。超える場合はタイトル+changed filesで絞り込む

## 実行フロー

```
Phase 1: PR収集 → Phase 2: 既存ルール/スキル読み込み → Phase 3: 分析 → Phase 4: 提案出力
```

---

## Phase 1: PR収集

引数 `$ARGUMENTS` を日数として解釈する（デフォルト: 7）。

```bash
# マージ済みPRを取得（日付フィルター付き）
gh pr list --state merged --limit 20 \
  --search "merged:>=$(date -v-${DAYS}d +%Y-%m-%d)" \
  --json number,title,mergedAt,labels,files,body
```

各PRについて以下を取得:

- **メタデータ**: PR番号、タイトル、マージ日、ラベル
- **変更ファイル一覧**: `gh pr diff <number> --name-only`
- **diff**: `gh pr diff <number>` で差分を取得

### diff サイズ制御

1. まず `gh pr diff <number> --name-only` でファイル数を確認
2. changed files が 30 以下なら全 diff を取得
3. 30 超の場合は `--name-only` の結果 + 主要ファイルの diff のみ取得
   <!-- 自動生成ファイルやロックファイルは除外 -->

---

## Phase 2: 既存ルール/スキル読み込み

以下のファイルをすべて読み込み、分析の参照基盤とする。

- `.claude/rules/` 配下の全ルールファイル
- `.claude/skills/*/SKILL.md` 配下の全スキルファイル

---

## Phase 3: 分析

収集したPRデータと既存ルール/スキルを照合し、以下の2つの観点で分析する。

### 観点1: 新パターンの検出

**目的**: 複数PRで繰り返し出現するが、既存ルール/スキルに未記載のパターンを発見する

**分析手順**:

1. 全PRの diff を横断的にスキャンし、繰り返し出現する以下を探す:
    - **コンポーネント構造パターン**: 新しいUI構成の定着
    - **API統合パターン**: 新しい呼び出し方、エラーハンドリング
    - **import パターン**: 新しいライブラリ/ユーティリティの定着した使い方
    - **命名規約**: ファイル名、変数名、型名の新しい慣例
    - **テストパターン**: 新しいテスト手法
    - **状態管理パターン**: 新しい状態管理の使い方

2. 「繰り返し」の基準: **2つ以上のPR** で同じパターンが出現

3. 既存ルール/スキルと照合し、**未記載のもの** のみを抽出

### 観点2: ルールドリフト（乖離）の検出

**目的**: 既存ルールに反するコードがマージされている場合、ルール更新 or 違反修正を提案する

**分析手順**:

1. 各PRの diff を既存ルールと照合
2. 乖離が見つかった場合、以下を判定:
    - **ルールが古い**: PRの方が正しく、ルールを更新すべき → ルール更新提案
    - **コードが違反**: ルールの方が正しく、コードを修正すべき → 違反報告
    - **例外ケース**: 特定の状況では既存ルールが適用できない → ルールに例外を追記

---

## Phase 4: 提案出力

分析結果を以下のフォーマットで出力する。

```markdown
# PR学習レポート

**分析期間**: YYYY/MM/DD 〜 YYYY/MM/DD
**対象PR数**: N件

## 分析対象PR

| # | タイトル | マージ日 | 変更ファイル数 |
| --- | --- | --- | --- |
| #123 | feat: ... | 2026-02-10 | 5 |

---

## 新パターン検出

### [パターン名]

- **検出PR**: #123, #125
- **出現回数**: N回
- **概要**: [パターンの説明]
- **提案アクション**: [新規ルール追加 / 既存スキル拡張]
- **対象ファイル**: `.claude/rules/[name].md` or `.claude/skills/[name]/SKILL.md`

<details>
<summary>コード例</summary>

[PRから抽出した実際のコードパターン]

</details>

---

## ルールドリフト

### [ルールファイル名]

- **違反/乖離PR**: #124
- **該当ルール**: [ルールの要約]
- **実際のコード**: [乖離箇所の要約]
- **判定**: ルール更新 / コード修正 / 例外追記
- **提案内容**: [具体的な修正案]

---

## サマリー

| 観点 | 件数 |
| --- | --- |
| 新パターン | N件 |
| ルールドリフト | N件 |
| **合計提案数** | **N件** |
```

### 出力ルール

- 提案がない場合は「検出なし」と明記する
- 各提案には **具体的なアクション** を含める
- コード例は実際のPR diff から引用する
- 優先度の高い提案を先に出力する

---

## 実行の効率化

### サブエージェント活用

PR数が5件以上の場合、Agent ツールで並列にdiffを収集する。

### コンテキスト節約

- 自動生成ファイルの diff は無視
- ロックファイルの diff は無視
- `.claude/` 配下の変更はルール/スキル自体の更新なので分析対象外

---

## 分析時の注意事項

1. **ボイラープレートを無視**: 自動生成ファイル、フォーマット変更のみのdiffは分析対象外
2. **1回限りのパターンは慎重に**: 1つのPRでしか出現しないパターンは、特に重要でない限りスキップ
3. **既にルール化済みのパターン**: 既存ルールで十分カバーされているパターンは報告不要
4. **chore/docs/ci PRはスキップ**: ルール/スキル自体の更新PRやCI設定PRはパターン分析の対象外
