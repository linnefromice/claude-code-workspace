# Santa Loop

santa-methodスキルを使用した敵対的デュアルレビュー収束ループ。2つの独立したレビューアー — 異なるモデル、共有コンテキストなし — が両方NICEを返すまでコードはリリースされません。

## 目的

2つの独立したレビューアー（Claude Opus + 外部モデル）を現在のタスク出力に対して実行します。両方がNICEを返すまでコードはプッシュされません。いずれかがNAUGHTYを返した場合、フラグされたすべての問題を修正し、コミットし、新しいレビューアーで再実行 — 最大3ラウンド。

## 使用方法

```
/santa-loop [file-or-glob | description]
```

## ワークフロー

### ステップ 1: レビュー対象の特定

`$ARGUMENTS` からスコープを決定するか、未コミットの変更にフォールバック。

### ステップ 2: ルーブリックの構築

レビュー対象のファイルタイプに適したルーブリックを構築。すべての基準に客観的なPASS/FAIL条件が必要です。最低限含むもの: 正確性、セキュリティ、エラーハンドリング、完全性、内部一貫性、リグレッションなし。

### ステップ 3: デュアル独立レビュー

2つのレビューアーを**並列**で起動:

- **Reviewer A**: Claude Agent（常に実行）— Opus、完全なルーブリック付き
- **Reviewer B**: 外部モデル（Codex CLIまたはGemini CLI。どちらも利用できない場合はClaudeフォールバック）

各レビューアーはルーブリックの各基準をPASS/FAILで評価し、構造化されたJSON判定を返します。

### ステップ 4: 判定ゲート

- **両方PASS** → **NICE** — ステップ6（プッシュ）に進む
- **いずれかFAIL** → **NAUGHTY** — 両レビューアーのクリティカル問題をマージし、ステップ5に進む

### ステップ 5: 修正サイクル（NAUGHTYパス）

1. 両レビューアーのすべてのクリティカル問題を表示
2. フラグされたすべての問題を修正 — フラグされたもののみ変更、ドライブバイリファクタリングなし
3. すべての修正を1つのコミットにまとめる
4. **新しいレビューアー**でステップ3を再実行（前ラウンドの記憶なし）
5. 両方がPASSを返すまで繰り返し

**最大3イテレーション。** 3ラウンド後もまだNAUGHTYの場合は停止し、残りの問題を提示。プッシュしないでください。

### ステップ 6: プッシュ（NICEパス）

両レビューアーがPASSを返した場合:

```bash
git push -u origin HEAD
```

### ステップ 7: 最終レポート

```
SANTA VERDICT: [NICE / NAUGHTY (escalated)]

Reviewer A (Claude Opus):   [PASS/FAIL]
Reviewer B ([使用モデル]):   [PASS/FAIL]

一致:
  両方がフラグ:     [両方がキャッチした問題]
  Reviewer Aのみ:   [Aのみがキャッチした問題]
  Reviewer Bのみ:   [Bのみがキャッチした問題]

イテレーション: [N]/3
結果:           [PUSHED / ESCALATED TO USER]
```

## 注意事項

- Reviewer A（Claude Opus）は常に実行 — ツーリングに関係なく少なくとも1つの強力なレビューアーを保証。
- Reviewer Bにはモデルの多様性が目標。GPT-5.4またはGemini 2.5 Proが真の独立性を提供。Claudeのみのフォールバックはコンテキスト分離による価値を提供しますが、モデルの多様性は失われます。
- 各ラウンドで新しいレビューアーを使用し、前の発見事項へのアンカリングバイアスを防止。
- ルーブリックが最も重要な入力です。レビューアーがラバースタンプを押したり、主観的なスタイルの問題をフラグする場合は強化してください。
- プッシュはNICE後のみ — ループ中は決して行わない。
