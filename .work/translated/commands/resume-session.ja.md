# Resume Sessionコマンド

最後に保存されたセッション状態を読み込み、作業を行う前に完全に状況を把握します。
このコマンドは `/save-session` の対になるコマンドです。

## 使用タイミング

- 前日の作業を続けるために新しいセッションを開始する時
- コンテキスト制限により新しいセッションを開始した後
- 別のソースからセッションファイルを引き継いだ時（ファイルパスを指定するだけ）
- セッションファイルがあり、Claudeに進行前に完全に吸収させたい時

## 使用方法

```
/resume-session                                                      # ~/.claude/session-data/ の最新ファイルを読み込み
/resume-session 2024-01-15                                           # その日付の最新セッションを読み込み
/resume-session ~/.claude/session-data/2024-01-15-abc123de-session.tmp  # 現在のshort-idセッションファイルを読み込み
/resume-session ~/.claude/sessions/2024-01-15-session.tmp               # レガシー形式の特定ファイルを読み込み
```

## プロセス

### ステップ 1: セッションファイルを検索

引数がない場合: `~/.claude/session-data/` で最新の `*-session.tmp` ファイルを探します。

引数がある場合:
- 日付（`YYYY-MM-DD`）のように見える場合、`~/.claude/session-data/`、次にレガシーの `~/.claude/sessions/` でマッチするファイルを検索
- ファイルパスのように見える場合、そのファイルを直接読み込み

### ステップ 2: セッションファイル全体を読み込み

### ステップ 3: 理解を確認

構造化されたブリーフィングで応答:

```
SESSION LOADED: [ファイルへの実際のパス]
════════════════════════════════════════════════

PROJECT: [ファイルからのプロジェクト名/トピック]

WHAT WE'RE BUILDING:
[自分の言葉で2-3文のサマリー]

CURRENT STATE:
PASS: Working: [count] 件確認済み
 In Progress: [進行中のファイルリスト]
 Not Started: [計画されたが未着手のリスト]

WHAT NOT TO RETRY:
[すべての失敗アプローチとその理由をリスト — これは非常に重要]

OPEN QUESTIONS / BLOCKERS:
[ブロッカーや未回答の質問をリスト]

NEXT STEP:
[ファイルに定義されている場合は正確な次のステップ]

════════════════════════════════════════════════
続行する準備ができました。何をしますか？
```

### ステップ 4: ユーザーを待つ

自動的に作業を開始しないでください。ファイルに触らないでください。ユーザーが次に何をするか指示するのを待ちます。

---

## エッジケース

- **同じ日付に複数のセッション**: その日付の最新の変更ファイルを読み込み
- **セッションファイルが存在しないファイルを参照**: ブリーフィング中にこれを注記
- **セッションファイルが7日以上前**: ギャップを注記し通常通り進行
- **セッションファイルが空またはマルフォーム**: 報告して `/save-session` で新しいものを作成するよう提案

---

## 注意事項

- 読み込み時にセッションファイルを変更しない — 読み取り専用の履歴記録
- ブリーフィングフォーマットは固定 — 空であってもセクションをスキップしない
- 「What Not To Retry」は "なし" でも常に表示される — 見逃すには重要すぎる
- 再開後、ユーザーは新しいセッションの終わりに `/save-session` を再度実行したい場合がある
