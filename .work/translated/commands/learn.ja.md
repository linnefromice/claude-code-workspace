# /learn - 再利用可能なパターンを抽出

現在のセッションを分析し、スキルとして保存する価値のあるパターンを抽出します。

## トリガー

非自明な問題を解決したセッション中の任意の時点で`/learn`を実行。

## 抽出対象

以下を探す:

1. **エラー解決パターン**
   - どんなエラーが発生したか？
   - 根本原因は何だったか？
   - 何で修正したか？
   - 類似のエラーに再利用可能か？

2. **デバッグ技術**
   - 自明でないデバッグステップ
   - 機能したツールの組み合わせ
   - 診断パターン

3. **ワークアラウンド**
   - ライブラリの癖
   - API制限
   - バージョン固有の修正

4. **プロジェクト固有パターン**
   - 発見されたコードベース規約
   - 行われたアーキテクチャ決定
   - 統合パターン

## 出力フォーマット

`~/.claude/skills/learned/[pattern-name].md`にスキルファイルを作成:

```markdown
# [説明的なパターン名]

**抽出日:** [日付]
**コンテキスト:** [これが適用される状況の簡単な説明]

## 問題
[これが解決する問題 - 具体的に]

## ソリューション
[パターン/技術/ワークアラウンド]

## 例
[該当する場合はコード例]

## 使用タイミング
[トリガー条件 - このスキルを有効化すべき条件]
```

## プロセス

1. 抽出可能なパターンのセッションをレビュー
2. 最も価値のある/再利用可能な洞察を特定
3. スキルファイルを下書き
4. 保存前にユーザーに確認を求める
5. `~/.claude/skills/learned/`に保存

## ノート

- 些細な修正は抽出しない（タイプミス、単純な構文エラー）
- 一度限りの問題は抽出しない（特定のAPI停止など）
- 将来のセッションで時間を節約するパターンに焦点
- スキルは焦点を絞る - 1スキルに1パターン
